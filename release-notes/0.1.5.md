# 0.1.5

A big batch of bug fixes and UI polish from a review pass.

## Bug fixes

- **Deleting a live fork target no longer "un-deletes" itself.** Deleting a
  property an active fork writes to (e.g. `amount-fork-as-links`) used to be
  recomputed from the source within a beat, so the deletion — and reverting it —
  never stuck. Deleting such a property now opens a fork-aware dialog: **Pause
  sync & delete** (keeps the rule, restorable) or **Remove rule & delete** (moves
  the rule to the restorable removed-list). Covers property/value/file scope.
- **The history tab refreshes in realtime.** A delete or find & replace in another
  tab now updates the history list (and its drift badges) without reopening it.
- **Notes open in the view's own window.** Opening a file from a popped-out plugin
  view now lands the tab in that popout window instead of jumping to the main one
  (Format Doctor links, Property Index open buttons). The sidebar-docked index
  still opens notes in the main editor area.

## Views: refresh-on-refocus + auto-return-focus

- All five views (Property Index, Find & Replace, History, Format Doctor,
  Conditional Formatting) re-render when their tab regains focus, fixing stale or
  blank deferred-leaf state.
- Opening a note from a view and then closing it returns focus to the view.

## Property Index

- Fuzzy filter: "fruit basket" matches a `fruit-basket` property (whitespace
  tokens, all must match, `-`/`_`/space treated the same).
- A clear (X) button on the filter, and an expand chevron on each property header.
- Property and value rows now use the same icons in the same order (open ·
  replace · rename · copy · delete); open tooltips read "Open N file(s) in new
  tabs with this property/value name".
- Expanded file rows swap their redundant trash for a "⋯" file menu (open in
  tab/right/below); the value keeps its trash. The value right-click menu drops
  its long inline file list and keeps Open-all · Copy · Rename · Delete.

## Format Doctor

- Each row shows a live **output preview** (current → result) so you can see what
  a value becomes before applying; empty inputs read "(skipped)", unparseable
  ones "(invalid)".
- **Select all / Deselect all** buttons (select-all skips still-blank
  needs-a-human rows).

## Allowed values

- The pin icon turns a clear warning **red** when a pinned property has values
  outside its allowed list, and a persistent notice offers a one-click **Open
  audit**.
- The audit command is renamed "Audit pinned allowed values" for discoverability.
- In the audit, each offending value now has an **"Add to allowed"** button next
  to "Find & replace" — adopt the value into the property's allowed list instead
  of replacing it, and it drops off the violations right away.

## Find & Replace history

- Renamed to **Property & Value History** (command: "Property, value & frontmatter
  history"); the "Force-revert drifted files" toggle moved to its own line.

## Also

- Post-deploy reload notices carry a **Reload now** button (dev tooling).
