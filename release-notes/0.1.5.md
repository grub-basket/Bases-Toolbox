# 0.1.5

Big-batch fixes and polish from a review pass.

## Bug fixes

- **Deleting a live fork target no longer "un-deletes" itself.** When you delete a
  property that an active fork writes to (e.g. `amount-fork-as-links`), live sync
  used to recompute it from the source within a beat — so the deletion (and any
  revert of it) appeared not to stick, and the history/JSONL audit looked wrong.
  Deleting such a property now opens a fork-aware dialog first: **Pause sync &
  delete** (keeps the rule, restorable) or **Remove rule & delete** (moves the rule
  to the restorable "removed" list), so the delete actually holds. Covers
  property-, value-, and file-scoped deletes.
