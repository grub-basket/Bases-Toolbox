# 0.1.43

## Add formula column (new)
- A new command and launcher tool, **"Add or fix a base formula column"**, adds
  a computed (formula) column to a base without hand-editing YAML. Pick a name
  and a Bases expression (e.g. `file.ctime`), choose which view(s) to show it in,
  and it writes the `formulas:` entry into the `.base` file and adds the column
  to the view's order.
- It also **repairs Obsidian's empty-formula glitch**: a formula created with a
  blank expression that the Bases UI locks you out of editing (and that survives
  reopening the base or restarting). Such a formula is flagged, and you can type
  an expression straight in to fix it, or remove it. The tool never writes an
  empty formula, so it can't create the glitch in the first place.
- Removing a formula also strips its `formula.<name>` column from every view.

## Native formula editing (autocomplete + validation)
- While the base is open, the expression fields — both when editing an existing
  formula and when adding a new one — become **Obsidian's own Bases formula
  editor**: autocomplete for functions and property/column names, plus inline
  syntax validation as you type. When the base isn't open, the field falls back
  to a plain text input.
- The in-app help now points to Obsidian's built-in formula editor for authoring
  and notes that the Formula Forge community plugin is complementary (reusable
  global functions and rendering formulas in note bodies).
