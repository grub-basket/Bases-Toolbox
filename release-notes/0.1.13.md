# 0.1.13

## Export — a lot more capable
- **From a base**: load a base, pick **which view** to export (bases can have
  several), and see that view's filters, columns, and the base's filters before
  exporting. Multi-folder bases are handled.
- **From a folder**: check off **subfolders to ignore**, and see the
  **non-markdown files** a CSV skips — one button **companions** them so they're
  included on the next load, regardless of your companion settings.
- **Column checklist** (both modes) — all on by default; pick exactly what to
  export.
- **Readable base summary (.md)** — a plain-language note of the base's filters,
  formulas, sort, and grouping, so you can rebuild in a spreadsheet what a flat
  CSV can't hold. The exporter now spells out these limitations inline.
- Folder box starts blank (blank = whole vault).

## Import
- **Drag and drop** a CSV/TSV onto the importer, or click the same zone to
  choose a file. Clearer supported-format text; binary spreadsheets are rejected
  with a helpful note. The **target folder autocompletes**.

## Reference
- **"Bases built-in properties"** section in the property index — click any
  `file.*` identifier (created time, tags, folder, …) to copy it, for when the
  Bases property menu "forgets" it.
- **Base embeds reference**: Settings → *Generate reference note* writes a
  packaged (offline) note documenting the embed display flags with examples.
