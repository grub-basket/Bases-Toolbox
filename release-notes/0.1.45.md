# 0.1.45

## Importer: paste a list, not just a table
- Records separated by **blank lines** are now detected as a list, with each
  record's lines becoming columns you name in the mapping table. Copying titles
  and URLs out of a browser tab-export extension, for example, pastes straight in
  as a two-column import — one note per record.
- A new **Input format** dropdown sets how the paste is read: **Auto** (default),
  **Table (CSV/TSV)**, or **List (blank-line records)**.
- The column count comes from the most common per-record line count, so an
  occasional stray line doesn't create a phantom column.

## Importer: URLs are no longer typed as links
- Obsidian's link property type only resolves internal `[[wikilinks]]`, so URL
  columns were importing as broken links. URL columns now import as **text**,
  leaving the address intact.
- The link type is still detected for genuine internal-link columns (a "link"
  column whose values are note names).

## Importer: set every column's type at once
- **"Set all columns to `<type>`"** applies one type across the whole mapping.
- **"Re-detect types"** restores the automatic detection — which also undoes a
  bulk change.

## Importer: progress, and a way into the result
- Long imports show a **progress bar** and a live **"Importing X/N…"** notice
  instead of appearing to hang.
- The **Import** button is disabled while a run is in flight, so it can't be
  fired twice.
- The completion notice has an **Open base** button that opens the base the
  import created.
