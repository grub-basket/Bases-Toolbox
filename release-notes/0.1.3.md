# 0.1.3

Property fork management, property-index file access, and conditional-formatting
per-base scoping.

## Property forks
- Live sync is now **on by default** when you fork a property, so forks appear
  as manageable rows in settings.
- Each fork can be **paused** (keeps the definition, stops recomputing) instead
  of only deleted; **deleting** moves it to a restorable "recently removed" list.
- Forks are **editable** — repoint the source/target property or change the
  transform without recreating the fork.
- A **"Fork builder…"** button in settings opens the convert/fork picker.
- If a fork's source or target property no longer exists (e.g. after renaming a
  property in the All Properties view), the fork row shows a **warning** so a
  broken fork is easy to spot and fix or pause.

## Property index
- Expand any value to **list the files that use it**, and open them — click,
  cmd/ctrl-click for a new tab, an open button, or a right-click menu
  (Open / new tab / to the right / below).
- Per-value and per-property counts now spell out what they mean ("N files
  have this value").
- A toolbar button **opens the property index in a main tab**.

## Conditional formatting
- Each rule can apply to **all bases or a chosen subset** via a per-rule sheet
  picker.
