# 0.1.0 — first release

Quality-of-life tools for Bases and properties. Everything below landed in
this initial release, in roughly the order it was built.

## Find & replace, history, and reverts
- Find & replace property values across the vault: pick a property, replace
  one specific value everywhere or override all values at once; list
  properties replace the matched item in place (initially with a
  single-level undo, later superseded by the full history below).
- Operation history: every run is logged with per-file before/after values;
  entries can be reverted individually. A user-configurable history cap
  (default: no cap) and a clear-history action live in the settings.
- Find & replace becomes a main-area tab: matched files appear as a
  checklist with `current → new` preview per file, partial apply, and live
  validation of the replacement value ("existing value — matches will merge"
  vs "new value").
- History becomes a main-area tab with selective per-file revert, a
  force-revert toggle for files edited since the operation, "edited since"
  drift badges, `old → new` rows, and a provenance chip showing which
  feature produced each entry. An entry only counts as reverted once nothing
  from it remains in effect, so partial and forced retries always work.
- A memoized property cache (invalidated by metadata events) backs the
  index, find & replace, and audits, so repeated opens cost nothing.

## Number input guard
- Number properties — in the frontmatter panel and inside Bases views — no
  longer change value on ArrowUp/ArrowDown or the mouse scroll wheel.
- Optional digits-only typing swallows stray characters like `e`; `.`, `,`
  (locale decimal) and `-` still type fine. Both toggles default on.

## Bulk editing
- Bulk edit the properties of a base's current results — whatever the open
  view shows after filters and search. Five modes: set, set only if missing,
  append items, remove items, delete the property entirely. All revertible;
  reverting a bulk-create deletes the property again, reverting a delete
  restores it.

## Rollups
- Compute count / sum / average / min / max over the notes linking to (or
  linked from) each result of the open base, written into a real frontmatter
  property Bases can display and sort. One-shot, re-run to refresh,
  revertible. Only genuine numbers aggregate — `null`, empty strings, and
  booleans are skipped.

## Conditional formatting
- Color Bases rows by rules (first match wins): property, operator
  (equals / not-equals / contains / comparisons / empty), value, and a color
  from the theme palette — or any custom color via a picker, applied at the
  same subtle tint strength. `not-equals` on list values requires every item
  to differ.

## CSV import & export
- Import CSV/TSV as one note per row: per-column property names (Title Case
  by default), types (text / number / date / boolean / list / link), include
  toggles with select-all, filename column, sample values under each header,
  live preview with row navigation, an optional note-body template with
  `{{Column}}` placeholders, an omit-empty option, a collision policy
  (suffix / skip / overwrite) probed against both the batch and existing
  vault files, and an optional auto-created `.base` over the imported
  folder. Dates normalize from US, European, written-month, and Excel-serial
  formats, with a warning counting ambiguous `M/D` vs `D/M` dates. Quoted
  fields may contain newlines. (Ported from the companion
  CSV-to-Obsidian-Properties-for-Bases web tool, then re-synced with its v2
  update.)
- Export the open base's results as clean CSV — wikilinks unwrapped, lists
  joined, spreadsheet formula injection neutralized — written next to the
  `.base` file and copied to the clipboard.

## Merge & duplicates
- Merge one note into another, frontmatter-aware: missing properties copy
  over, list properties union, scalar conflicts get a per-property picker.
  The source's body is appended, every link to the source — body links and
  frontmatter links — is re-pointed at the target with aliases preserved,
  and the source moves to the vault trash (recoverable).
- Find duplicate notes by similar file names (case, punctuation, and
  copy/number suffixes ignored), by equal values of a chosen property,
  and/or by identical bodies (verified byte-for-byte within hash groups).
  Pick the keeper per group; merging requires a confirming second click.

## Property index & allowed values
- Property index sidebar: every frontmatter property in the vault with type,
  file count, and per-value usage counts; search-filterable; per-property
  shortcuts into find & replace, a copy-name button, a button that opens the
  core All Properties view pre-filtered, and a pin button.
- Pinned allowed values: pin a property's permitted values; editing that
  property in the frontmatter panel or a Bases cell shows a picker; an audit
  command lists every out-of-list value with one-click find & replace jumps.

## Format doctor
- Property format doctor: finds every value that doesn't match its
  property's assigned type (the mismatches Obsidian flags but "fixes"
  destructively), explains what each type expects with an example, and
  pre-fills a safe suggested fix when one exists — dates normalized, `yes` →
  `true`, scalars → one-item lists, a date in a datetime slot gains
  `T00:00`. Inputs are validated before writing; invalid ones are marked and
  the file left untouched; empty means skip — a value is never erased.
  Applied fixes are revertible.

## Property fork / convert
- Convert or fork a property's format: normalize legacy dates to
  `YYYY-MM-DD`, unwrap `[[wikilinks]]` to plain text, or wrap plain text in
  wikilinks — in place, or forked into a second property so both formats
  coexist. Forks can be kept in live sync: editing the original recomputes
  the fork automatically. Active syncs are listed and removable in settings.

## Migration & display extras
- Inline-field migrator: converts Dataview-style `Key:: value` lines and
  `[key:: value]` spans into real frontmatter, with folder scoping, dry-run
  preview, typed values, optional body cleanup (off by default), and
  history-logged writes.
- Filter quick-toggle: disable individual `.base` filter conditions without
  deleting them; disabled filters are remembered and re-enable in place.
- Cell zoom editor: open a focused Bases cell or property value in a large
  editor modal; lists edit one item per line; nested objects are protected.
- Embedded-base display flags via embed alt text: `bases-no-toolbar`,
  `bases-no-header`, and `bt-height-N` for a fixed height.
- Multiline list cells: a toggle that stacks list-property pills vertically
  in Bases table cells (long lists scroll in-cell; pairs with the native
  row-height option).

## Reviews, hardening, and housekeeping
- Full code + security review before release: spreadsheet formula-injection
  escaping on export, exact-body verification in the duplicate finder,
  `__proto__`/`constructor` property names refused, a bounded inline-field
  scanner, plus seven correctness fixes — filter-toggle state desync, silent
  loss of duplicate filter conditions, rollups counting `null` as 0, Excel
  serial dates off by a day west of UTC, the comma decimal separator being
  blocked, bulk "set empty" creating null properties, and bracketed-field
  cleanup missing non-canonical spacing. Details in
  `docs/review-2026-07-02.md`.
