# 0.1.4

A large release: a much more capable Property Index (delete, rename, per-value
search, type icons, an audit trail), conditional-formatting autocomplete +
duplicate detection + a side panel, smarter property forks, companion-note
improvements, format-doctor fixes, and an in-app feature reference.

## Property index
- **Delete a property** at three scopes: from every file, from just the files
  with a given value, or from a single file. Every deletion is **undoable** from
  the find & replace history.
- **Deletion audit** — each removal is appended to a durable JSONL log (one file
  per property, under the plugin folder). A persistent, theme-aware notice with
  an **"Open export…"** button lets you open it in your default app, reveal it in
  Finder / File Explorer, or copy its path. **Restoring** a deletion prunes its
  rows back out of the log.
- **Rename a property** across all files (folds into an existing name when one
  exists), and **rename a value** across the files that hold it — both undoable.
- **Search by value** — the filter now matches a property by name *or by any of
  its values*; a value-only match auto-expands to show just the matching values.
- **Per-type icons** — each property shows a Lucide icon for its widget type
  (text, list, number, checkbox, date, datetime, tags, aliases, …).
- **Open files in new tabs** — clicking a file opens it in a new tab by default
  (alt-click reuses the current tab), plus "Open all N files in new tabs" for a
  property or a value, with a confirm past a threshold.
- **Copy value** button on each value.
- **Responsive header** — actions show inline when there's room (and always in
  the wide main-tab popout) and collapse into a **⋯** overflow menu when the
  panel is narrow, so nothing clips.

## Conditional formatting
- **Autocomplete** for property names and values while building rules — pulled
  from the full vault index, so the list is never truncated the way Obsidian's
  built-in property suggester is.
- **Duplicate detection** — adding a rule whose condition already exists is
  refused, and existing duplicates are flagged.
- **Side panel** — manage rules in a narrow-friendly panel that can pop out to
  its own tab. Suggestions and duplicate detection work there too.

## Property forks
- Forks are now **always listed** in settings after creation (with a live-sync
  toggle), and a **fix** for forks whose definition wasn't persisted when the
  initial transform failed.
- **Smart default names** based on the transform (e.g. `date-fork-normalized`),
  auto-updating as you change the transform.
- **Adopt detected forks** — fork-shaped properties that aren't yet managed are
  offered for one-click adoption so live sync can pick them up.
- An **"ignore warning"** option on a fork whose source/target looks missing.

## Companion notes
- The extension filter is now a **blacklist** (every non-Markdown file is
  eligible by default) with a **chip UI** — excluded types plus the types found
  in your vault, click to toggle.
- **Folder-scoped** auto companioning (with a whole-vault option, off by
  default), **retroactive** companioning when you enable auto mode, and `.edtz`
  version files are always ignored.

## Property format doctor
- Applying checked fixes no longer **collapses the list** to a partial set — the
  view waits for Obsidian's metadata cache to settle, and now **self-corrects**
  as the cache resolves so opening/rescanning always shows the full set.

## Settings
- An in-app **reference** at the bottom of settings lists every command, panel,
  and automatic feature — no need to leave Obsidian to discover what's here.
- Property forks get their own settings heading; clearer "Add formatting rule"
  label.
