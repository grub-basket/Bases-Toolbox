# 0.1.2

New features for non-Markdown files and metadata durability, a settings/command
consistency pass, and a substantial conditional-formatting overhaul.

## Companion notes for non-Markdown files
- New command **Create companion notes for non-Markdown files**: Bases can't
  read frontmatter from images, PDFs, audio, etc., so this creates a small
  companion note per file (`<file>.<ext>.md`) carrying the file's metadata as
  frontmatter — `file-name`, `file-ext`, `file-size`, `file-created`,
  `file-modified`, `file-path` — plus a `companion-of` link and an embed of the
  file. Companions live adjacent to their files by default, or in a designated
  folder that mirrors the source structure.
- Scan-first preview shows how many companions will be created vs refreshed.
  Re-running refreshes the `file-*` properties and never touches properties you
  added. Files derived from notes (e.g. version files other plugins create) are
  skipped so companioning can't cascade.
- Settings section with a default destination, an extension filter, and an
  opt-in **auto mode** that companions newly added files (armed after startup so
  the launch-time file-scan isn't mistaken for a bulk drop).

## Stamp file metadata into note properties
- New command **Stamp file metadata into note properties**: filesystem
  created/modified times get destroyed by syncs, copies, and migrations, while
  frontmatter travels with the note. This snapshots each Markdown note's current
  times into properties you name (default `created` / `modified`), scoped to a
  folder or the whole vault. Set-if-missing by default; overwrite is opt-in.
  Logged to history and revertible.

## Conditional formatting overhaul
- **Row or cell scope per rule**: color the whole row (default) or just the
  matching property's cell.
- **Real-time updates**: rule changes now apply to open bases without an app
  reload — including bases in popout windows. (Previously a new rule often
  didn't show until reload.)
- **Rules are fully editable inline** — property, operator, and value can now be
  changed on an existing rule, not just its color/enabled state.
- **Reorder rules** with up/down controls (rules apply top-to-bottom; first
  match wins per row and per cell).
- **Color swatch preview** on each rule; **capitalized color names**
  (Red / Orange / … / Custom…).
- **Custom color picker** no longer renders clipped; **dropdowns size to their
  selected text** instead of a fixed-wide box; the **"Add rule" label** no
  longer truncates.

## Settings & commands consistency
- **Pinned allowed values** now have a settings section listing every pinned
  property with a remove control (previously createable only from the property
  index, with no way to review or remove them in settings).
- **Toggle commands** for the number guard, digits-only typing, and multiline
  list cells — so they can be bound to hotkeys.
- New commands **Open Bases Toolbox settings** and **Open conditional formatting
  rules** (the latter deep-links to the section, scrolls it into view, and
  flashes it) — conditional formatting previously had no command entry point.
