# 0.1.6

Improvements to the pinned allowed-values audit and the pin indicator.

## Pinned allowed-values audit
- The audit now opens with a **"Pinned properties"** overview listing *every*
  property that has a pin — compliant ones ("all values OK") and violating ones
  ("N outside the list", flagged) — each with an **Edit pinned values** button,
  so you can see and manage all your pins in one place instead of hunting the
  property index for pin icons.
- Each value outside its allowed list has an **"Add to allowed"** button next to
  "Find & replace" — adopt the value into the property's allowed list instead of
  replacing it, and it drops off the violations right away.
- Property group headers are flush-left with a property icon (no more odd
  indentation).

## Pin indicator
- The pin icon has three clear states: **muted** (not pinned), **accent**
  (pinned, all values within the list), and **red** (pinned, some values
  outside). The accent now applies reliably.
- The violation toast and the red pin icon update **immediately** after a pin
  change (add to allowed, save, or clear) — before, they only refreshed on a
  vault change and could linger stale. While the audit modal is open the toast
  stays hidden, since you're already resolving violations there.
