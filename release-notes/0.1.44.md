# 0.1.44

## Read-only bases (new)
- Lock a base so its cells can't be edited — and, because the row body stops
  receiving pointer events, rows can't be accidentally right-click-deleted
  either. Internal/external links and the date-picker stay clickable.
- Two ways to choose what's locked, under **Settings → Read-only bases**:
  - **Make all bases read-only** — a global switch.
  - A **per-base list** — lock individual bases (with an "Add a base…" picker and
    a remove button; a base that no longer exists is flagged).
- Commands **"Toggle read-only for this base"** and **"Toggle read-only for all
  bases"**, plus a launcher tool.
- A separate opt-in, **"Also prevent adding rows,"** additionally hides the
  toolbar **New** button on read-only bases — for teams that want a hard lock
  rather than an append-only base. Off by default.
- Applied by marking each base's view so the lock survives Bases re-rendering its
  table, and kept in sync as bases open and close.
