# 0.1.16

Store-review compliance plus a few table/property polish items.

- **Store-review lint fix.** The one `eslint-disable` directive in the code now
  carries its rationale inline (via the `--` separator), which the Obsidian
  community-store review requires. No behavior change.
- **Filter quick-toggle keeps a stable order.** Toggling a filter on/off no
  longer re-ranks the list (enabled-over-disabled), so rows don't jump under the
  cursor and you can't accidentally click the wrong one.
- **Property index — rename value box spans the modal**, and value inputs now
  support `[[` internal-link autocomplete when renaming a value.
- **Multiline list cells — one-time tip.** Enabling the setting now shows a
  brief pointer to Bases' own row-height option, since stacking the list values
  doesn't grow the (virtualized) row height on its own.
