# 0.1.11

## Safer, clearer reverts (History)
- **Per-entry data-loss warning.** Each history entry now shows a red warning
  after its name *only when reverting it could lose data*:
  - **Merges** — always (reverting restores the whole note, overwriting any
    edits you made after the merge).
  - **Everything else** — only when "Also revert notes I've edited since the
    change" (force) is turned on. With it off, those notes are safely skipped,
    so no warning.
  The warnings appear and disappear live as you toggle that option.
- **Skipped files are shown after a revert.** Instead of a fleeting notice, the
  entry now lists every file that was *not* reverted — each clickable to open —
  tagged with why: edited since (left untouched to protect your newer edit),
  property renamed/removed, file moved/deleted, or (for merges) a note now
  occupies that path. When drift caused the skip, a note explains how to force
  it if you really want to.
