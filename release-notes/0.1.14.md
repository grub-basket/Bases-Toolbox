# 0.1.14

## Edit / create note properties (roomy form)
- **New note with properties** and **Edit properties of the active note** open a
  proper form for a note's frontmatter — a comfortable alternative to Obsidian's
  cramped inline / Bases-popup editor. Type-aware value widgets
  (text / number / checkbox / date / list / tags / aliases), add/remove rows,
  and a **File name** field that renames the note (the file-name editing that
  zoom-into-cell can't do).
- Reachable from two commands, a note's right-click menu, and the launcher.
- **Internal links**: type `[[` in a text or list value to autocomplete a note
  and insert a wikilink — it keeps working across multiple links in a list.
  A 🔗 button also inserts one. (Tags and aliases don't offer links.)
- **From a base**: launched while a base is focused, it borrows that view's
  editable columns and the base's folder, so the note slots straight in — like
  adding a row. Otherwise it opens with a few blank rows (count configurable in
  settings).

## Launcher favorites
- Star any feature to pin it to a **★ Favorites** section at the top; **drag to
  re-arrange**. "New note with properties" is pinned by default.

## Conditional formatting
- The **"is duplicated"** rule now re-evaluates in **real time** — it no longer
  needed a reload to catch a value becoming (or ceasing to be) a duplicate.

## Companions
- `.base` files are excluded from companioning by **default** (a one-time
  change you can undo by removing "base" from the exclude list). `.edtz` stays
  hard-excluded.

## Smaller
- CSV export/import folder & base pickers are wider, so the hint/path is
  readable.
- CSV export "From a base" writes a readable `.md` summary and lists what a flat
  CSV can't hold; the folder picker starts blank; column checklist per export.
