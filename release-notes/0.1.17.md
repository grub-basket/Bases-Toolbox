# 0.1.17

Bug fix: two features didn't detect an open base unless it was the *focused*
leaf.

- **"New note with properties" now borrows the base's columns reliably.** When a
  base was open but focus was elsewhere (a cell editor, the command palette, a
  side pane), the modal fell back to blank rows instead of the base's columns.
  It now finds the open base the same way Bulk Edit does, so the new note slots
  into the base as intended.
- **"Zoom into cell" no longer says "no base is open"** when a base is open but
  not the active leaf. It now resolves the `.base` from the leaf that actually
  contains the cell you clicked, instead of the (possibly different) active file.

Both relied on `activeLeaf` / `getActiveFile()` pointing at the base, which
isn't guaranteed; they now use `getLeavesOfType("bases")` as the reliable
signal. Verified live with a base open behind another focused view.
