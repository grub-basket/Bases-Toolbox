# 0.1.18

Extends the 0.1.17 base-detection fix to the remaining features, so nothing
depends on the base being the *focused* leaf.

- **Export active base results as CSV** now works when a base is open but not
  focused (it previously errored with "Focus a base first").
- **Allowed-values picker on Bases cells** resolves the right `.base` from the
  cell you clicked, instead of the active file.
- **Bulk edit / Compute rollup** now prefer the on-screen base when several are
  open, rather than picking one blindly.

Under the hood these all share one detection helper now (`base-detect.ts`),
which prefers the focused base, then the visible base, then the sole open base —
so behavior is consistent across every command. No change when the base already
has focus.
